# 渲染优化知识体系

## 核心问题

**渲染性能瓶颈**：CPU渲染指令准备速度 > GPU处理速度 → 帧率下降

---

## 一、GPU性能优化

### 1.1 问题根源

**双缓冲/多缓冲机制**缓解CPU-GPU不匹配：
- 旧方案：增加缓冲帧数（但增加输入延迟）
- 新方案：优化GPU负载

### 1.2 Shader优化

#### 简单vs复杂Shader性能差异

| 维度 | 简单Shader | 复杂Shader |
|------|-----------|-----------|
| 计算量 | 小 | 显著增加 |
| 特效 | 基础颜色/透明度 | 粒子/光影/物理 |
| 帧率影响 | 流畅 | 掉帧 |

#### 掉帧原因

- **渲染超时**：超过16.6ms/帧（60FPS）
- **特效叠加**：负载倍增
- **透明特效**：像素重复绘制

#### 优化方案

**精简Shader逻辑**：
- 减少纹理采样次数
- 避免动态计算
- 使用固定功能管线替代复杂计算

**控制特效层级**：
- 限制粒子数量
- 合并同类特效
- 控制透明层级

**动态降级机制**：
- 根据帧率自动调整特效质量
- 高帧率：全特效
- 中帧率：减少粒子/简化Shader
- 低帧率：只保留最基本效果

## 二、CPU性能优化

### 2.1 CPU瓶颈类型

**Draw Call瓶颈**：
- Draw Call过多导致CPU成为瓶颈
- 每个Draw Call都有CPU开销（序列化、状态切换）

**动画系统瓶颈**：
- 复杂角色动画计算量大
- 骨骼动画逐帧处理

### 2.2 PMU事件分析

**PMU（性能监控单元）**：从"猜"到"证"的跃迁

| 传统方式 | PMU方式 |
|---------|---------|
| "可能是路太窄" | "路口80%时间等左转" |
| "可能是车太多" | "货车把桥压满" |
| "可能是红绿灯坏了" | "导航系统算错路线" |

**应用场景**：
- SoA布局优化
- 分桶策略
- 减少虚调用
- False sharing处理
- 批处理与快照化

## 四、大型活动优化案例（《天刀》）

### 4.1 客户端策略

**同屏限流**：
- 限制同时渲染的角色数量
- 高负载场景自动降低数量

**特效分级**：
- 近处：全特效（高质量）
- 中距离：减少粒子/简化Shader
- 远处：只保留轮廓/基础效果

**UI分级预算**：
- 高负载场景：简化UI动画
- 关键信息优先显示
- 减少UI重绘

### 4.2 服务器策略

**分流**：
- 多区域分流玩家
- 热点区域玩家引导至其他区域

**限流**：
- 限制同时在线玩家数
- 超过阈值时拒绝新连接

**降级保命**：
- 关闭非核心功能
- 降低同步频率
- 简化AI逻辑

### 4.3 联动策略

**负载等级统一指挥**：
- 客户端报告负载等级
- 服务器调整活动强度
- 双方策略一致

**负载等级划分**：
- Level 1: 正常（全特效）
- Level 2: 轻度（减少20%特效）
- Level 3: 中度（减少50%特效）
- Level 4: 重度（只保留核心功能）

### 4.4 运营策略

**活动分散**：
- 多时间段活动
- 分散玩家高峰

**错峰补偿**：
- 高峰期玩家补偿
- 降低奖励获取难度

### 4.5 工具保障

**压测**：
- 模拟高并发场景
- 验证限流降级效果

**监控**：
- 实时监控服务器负载
- 客户端性能监控

**演练**：
- 定期进行故障演练
- 验证应急预案有效性

## 五、同屏性能优化（《天刀》100人同屏）

### 5.1 CPU瓶颈分析

**Draw Call瓶颈**：
- 角色数量多 → Draw Call多
- 解决：合并Draw Call、批处理

**动画系统瓶颈**：
- 每个角色都有动画状态机
- 解决：LOD动画、动画合并

**CPU时间分布**：
```
角色动画: 40%
Draw Call: 30%
物理计算: 20%
其他: 10%
```

### 5.2 GPU瓶颈分析

**阴影SSAO**：
- 阴影计算量最大
- 解决：降低阴影质量、使用缓存

**角色自发光/特效**：
- 高亮角色需要额外处理
- 解决：分级渲染

**头发布料**：
- 头发材质复杂
- 解决：LOD材质、简化模型

**远处角色**：
- 远处角色细节不重要
- 解决：只保留基础形状

### 5.3 UI影响

**远处角色动画**：
- 远处角色动画影响小
- 解决：减少远处角色动画更新频率

**帧时间变化**：
- 记录每关一项的帧时间变化
- 谁降得最多谁是大头

### 5.4 分级策略

**渲染LOD**：
- LOD0: 近距离，全细节
- LOD1: 中距离，减少细节
- LOD2: 远距离，只保留轮廓

**逻辑LOD**：
- LOD0: 完整AI逻辑
- LOD1: 简化AI逻辑
- LOD2: 只保留移动

**UI LOD**：
- LOD0: 完整UI
- LOD1: 简化UI
- LOD2: 只显示关键信息

**最有效优化**：
- 不是"再快一点"
- 而是"谁该认真画、谁该随便画；谁该认真算、谁该少算"

## 六、游戏引擎定制化优化

### 6.1 核心思想

**砍掉用不到的通用功能**

### 6.2 优化效果

**稳帧**：
- 减少不必要的计算
- 降低内存占用

**加载**：
- 减少不必要的资源加载
- 优化资源打包

**包体**：
- 减少不需要的代码
- 压缩不需要的资源

### 6.3 实施步骤

1. **功能审计**：
   - 列出所有引擎功能
   - 标记已使用/未使用

2. **禁用未使用功能**：
   - 关闭未使用的模块
   - 移除未使用的代码

3. **优化已使用功能**：
   - 精简实现
   - 使用更高效的算法

4. **测试验证**：
   - 功能测试
   - 性能测试
   - 兼容性测试

## 九、优化原则

1. **数据驱动**：用数据说话，别猜
2. **优先级明确**：先优化大头，再优化细节
3. **分级策略**：不是所有人都全特效
4. **动态降级**：根据性能自动调整
5. **监控告警**：实时发现性能问题
6. **门禁机制**：防止性能回归
7. **定期复盘**：持续改进
